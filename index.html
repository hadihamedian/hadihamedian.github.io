Selected Projects

ExchangeRateProvider

Problem

Financial institutions require aggregated, real-time exchange rate data from multiple vendors with failover capabilities to ensure zero downtime and data accuracy.

Solution

Architected a distributed rate aggregation system using .NET Core and RabbitMQ. Implemented a configurable aggregation engine that normalizes data streams from diverse providers. Designed dual-interface access patterns: highly optimized WebSockets for real-time tickers and REST endpoints for historical reporting, secured via JWT.

Outcome

Achieved sub-second data propagation and 99.99% system availability. The architecture supports dynamic provider integration without deployment, serving as the central pricing authority for downstream trading applications.

API Documentation & Testing SDK

Problem

Backend teams struggle with maintaining up-to-date API documentation and enabling easy consumer testing without leaving the development workflow.

Solution

Developed a drop-in .NET SDK (internal brand: "OpenDuckly") that auto-generates interactive documentation from code annotations. Built a full testing suite within the UI, allowing consumers to execute requests against live API endpoints directly from the docs. Engineered for zero-config integration into existing ASP.NET Core pipelines.

Outcome

Adopted across the organization as the standard for API deliverability. drastically reduced integration support time by empowering frontend teams and third-party consumers to self-serve via accurate, executable documentation.
